# 工作流

## 总流程

```text
客户提交问题
→ 描述修正
→ 附件解析
→ 等待两个步骤均完成
→ 补充判断与预回复生成
→ 客服查看并按需修正
→ 确认复制
→ 客服手动发送
→ 按路径更新状态
```

## 信息不足路径

```text
待分析
→ 输出缺失信息项
→ 生成补充信息话术
→ 客服查看并按需修正
→ 确认复制
→ 手动发送补充话术
→ 待补充信息
```

客户补充资料后，流程重新进入“待分析”。

## 信息充分路径

```text
待分析
→ 生成对外预回复
→ 待发送确认
→ 客服查看并按需修正
→ 确认复制
→ 手动发送正式回复
→ 已完成
```

## 同步约束

仅在“描述修正状态 = 已完成”且附件解析已有明确状态时，触发补充判断和预回复生成。这样可以避免重复触发和使用未完成数据。
