# Prompt 结构

公开仓库只提供任务结构和约束，不包含内部专有 Prompt 原文。

## 描述修正

- 目标：纠错、规范化表达。
- 约束：不新增事实；保留链接、路径、编号、错误码、文件名和版本号；原文清晰时原样返回。
- 输出：修正后的描述文本。

## 附件解析

- 目标：提取可用于判断问题的证据。
- 约束：区分无附件、解析成功、格式不支持和解析失败。
- 输出：状态、摘要、结构化结果和错误说明。

## 补充判断与预回复

- 目标：判断信息完整度，并生成客服可复制文本。
- 约束：对外回复不得包含思考过程；未命中已验证知识时提示人工判断；不得输出未经验证的确定性结论。
- 输出：缺失信息、内部判断依据、补充话术或对外预回复。

## 最终回复整理

- 目标：根据客服修正内容生成正式回复。
- 建议结构：可能原因、解决方案、参考教程。
- 约束：高风险问题需要明确建议升级人工处理。
