# Ticket Copilot：客户问题标准化与回复生成器（MVP）

将客户的一句话问题整理为可复制发送的客服回复，并在客服确认后沉淀最终版本。

本项目提供公开版流程、字段规范、Prompt 结构、脱敏样例和可复用 Skill。它不包含任何内部自动化配置，也不实现自动发送。

## 核心原则

1. AI 预回复只用于分流和提速，不自动发送。
2. 所有回复必须由客服查看并按需修正。
3. 客服点击“确认复制”后，手动粘贴并发送。
4. 人工确认是固定主流程，不是异常分支。
5. 对外预回复不得包含模型思考过程。
6. 规则优先于 AI，AI 不得自行改状态。

## 主流程

```text
客户提交问题
→ 描述修正
→ 附件解析
→ 等待描述修正和附件解析均完成
→ 判断是否缺少信息
→ 生成补充话术或对外预回复
→ 客服查看并按需修正
→ 点击“确认复制”
→ 客服手动发送
→ 按路径更新状态
```

信息不足时，客服确认并发送补充话术，状态进入“待补充信息”。  
信息充分时，系统生成对外预回复，客服确认并发送正式回复，状态进入“已完成”。

## 固定状态

```text
待分析 / 待补充信息 / 待发送确认 / 已完成
```

## 固定附件状态

```text
无附件 / 解析成功 / 格式不支持 / 解析失败
```

## 快速开始

1. 阅读 [项目说明](docs/01-project-overview.md) 和 [工作流](docs/02-workflow.md)。
2. 按 [字段规范](docs/04-field-spec.md) 在你的工单系统中创建最小字段集。
3. 按 [触发矩阵](docs/05-trigger-matrix.md) 配置流程。
4. 根据 [Prompt 结构](docs/06-prompt-structure.md) 适配你使用的模型。
5. 使用 [演示案例](docs/09-demo-cases.md) 做验收。
6. 如需让 Agent 协助搭建或检查流程，使用 [Skill 说明](skill/SKILL.md)。

## 项目边界

公开仓库不包含：

- 自动化 ID 或字段 ID
- 账号、Token、密钥
- 内网地址
- 真实客户数据和附件
- 内部专有 Prompt 原文
- 自动发送能力

详细说明见 [发布边界](docs/10-release-boundary.md)。

## 版本规划

- `v1.1.0`：知识库增强，包含标准话术库、知识命中记录、教程引用和知识沉淀规则。
